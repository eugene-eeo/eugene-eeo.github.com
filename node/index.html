XMLRPC Node

Cluster-Based Deployment

The XMLRPC Node is a simple (made of three scripts) and robust implementation of both the BareMetal Node concept and meant as a protocol for anyone's dreams of implementing their own Beowulf Cluster.

In short, Node is an XMLRPC protocol based "framework" for you to deploy commands, or tasks upon. Fortunately, Node does not have a massive API with complex commands- it's a relatively small and lightweight script designed to be as realistic and user-friendly as possible.

You can download the source from the following links:

    Node Server | Controller + Interface | Worker Node

Note: before you deploy your own cluster and perform whatever you want with it (i.e. pranks are a good idea) it's recommended that you actually read RFC01 for a basic overview of the Node concept and what it actually is. Documentation is currently via the developer's understanding of the comments placed inside the code.
