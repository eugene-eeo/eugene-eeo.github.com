class express.helpers.Model(instance)

Model object serving as a flexible RPC-style blueprint for the underlying API. The schema defaults to:

{"status":"{status}", "value":"{value}"}

Which should be enough for most lightweight API purposes.

Model._run(string, params={}, status_value_key='status', return_value_key='value', additional={}, return_as_dict=True)

Will get the dictionary of available methods via Express.get("METHODS") and will then check if the specified API call is within the said dictionary, then execute it. I.e.:

app.set("METHODS") = {"hello":lambda: return "Hello World!"}

And you have a request with the following payload (assume default return schema with sensible assumptions):

{
  "method":"hello",
  "params":{}
}

You pass the method key to the _run function, along with parameters at param:

method = request.form["method"]
params = request.form["params"]

print(app.Model._run(method, params))

And you will get this on your display, but nothing on the client side:

{
  "status": true,
  "value": "Hello World!"
}

The params option can either be a list or a dictionary, but it doesn’t perform any validation (number of keys, types, etc.) for you- you must catch any exceptions and errors yourself.

For your custom schemas, you can configure the _run function with the following options:

  • status_value_key: defaults to “status”, is the key which you will use for the status key (true if executed, false if it didn’t).

  • return_value_key: defaults to “value”, is the key which is used for the return value of the fucntion.

  • return_as_dict: defaults to True since most REST-APIs use dictionaries as their primary form of data transport. You may want to specify a value that gets returned instead of the dictionary via changing this value to something within the precompiled dictionary, i.e.:

    app.Model._run("hello",{},return_as_dict="value")
# => "Hello World!"
Model.functions

Sort of a private function to return the list of functions that we support within the Model. Not recommended to be used directly by explicit binding into the Express object, but you can do so:

express.helpers.Model(app).setup()

Which will unpack everything and load it into the main Express instance. The unpacked functions include:

  • _run
  • resp
  • schema
  • serve
Model.resp(params)

Public and private method for replying to the method calls with the user-defined schema. The schema should at least contain some remotely helpful information.

Your schema must also support the following keys during formatting, unless you have a special _run function:

  • value: the return value of the requested function, i.e. when the method 1 + 1 is called, the value is 2.
  • status: the status, True if the function was ran and False if it isn’t.
Model.schema(dictionary)

Overrides the current schema with the specified schema at the dictionary argument.

Model.serve(request)

This should be subclassed to another method because it will raise NotImplementedError by default since we do not want to make any assumptions.

The said function that overrides this must accept a new flask.request instance as it’s first argument, an object which the function can play around with.

You can explicitly set the serve function like this:

app.Model.serve = my_serve_function

Or you may want to subclass the Model class:

class MyModel(app.Model):
    def serve(self, request):
        # something here
Model.setup()

Sets up the Express instance by setting all of the functions in the Model.functions property with the Express instance. Code:

for item in self.functions:
    setattr(self.instance, item,
            getattr(self, item))
class express.helpers.Configuration(instance)

Adds configuration-based functions to the Express object, this is added upon runtime. All functions specified below are accessible directly from the Express object.

Configuration.disable(key)

Disable an application’s configuration value, i.e.:

app.disable("ENCODE-RESPONSES")
Configuration.disabled(key)

Checks if something is disabled, i.e.

app.disabled(“ENCODE-RESPONSES”) # => True
Configuration.enable(key)

Enable a configuration value:

app.enable("ENCODE-RESPONSES")
Configuration.enabled(key)

Checks if something is enabled, i.e.

app.enabled(“ENCODE-RESPONSES”) # => False
Configuration.functions

List of functions, mainly for internal use.

Configuration.get(key)

Get a configuration value, will raise KeyError if it is not present.

app.get(“ENCODE-RESPONSES”)
Configuration.maps(**args)

Overrides the configuration options of the current Express instance to the return value of the wrapped function, i.e.:

@app.maps()
def myconfig():
    return {}

But for more advanced apps something like this may be required:

@app.maps(dev=True)
def myconfig(dev):
    if dev:
        # development options
    else:
        # production options
Configuration.set(key, value)

Set a configuration value:

app.set(“ENCODE-RESPONSES”, True)
Configuration.setup()

Explicitly sets up the Express object with the functions defined within the Configuration class.

class express.helpers.Protocol(instance)
Protocol.deserialize(string)

Deserializes the incoming request contents or string via the options specified in ENCODING (in which it will decode the bytes object using the given charset) and STRIP-REQUESTS.

If STRIP-REQUESTS is enabled (is set to True), the incoming data will be ``.strip()``ed which may enable faster deserialization.

Protocol.escape(string)

Escapes the string using the options specified in ESCAPE-CHARSET, in the following format:

{"character-to-be-replaced":"replacement"}
Protocol.functions

List of functions that should be integrated with the main Express() instance.

Protocol.serialize(obj)

Serializes the presumably outgoing object via the serializer, will encode the outgoing string with the user’s encoding of preference.

Protocol.setup()

Sets up the Express environment or any environment in particular with a get and set API.

Protocol.unescape(string)

Unescapes the string according to the options specified in ESCAPE-CHARSET, then returns the escaped string.

class express.helpers.DB(instance)

DB object which can be used to bind everything within the object to the DB object itself. The new object (the one that is binded) can be obtained via DB.object.

Note that the DB object cannot simply bind to anything, but must bind to objects that support an Express instance as their first argument. Recommended for use in dynamic binding in which this object acts as a proxy.

DB.functions

Lists the functions in which the DB object will be assigned to. In this case we will only allow the “use” function.

DB.use(obj, **args)

Act as the proxy for the specified object, with optional arguments specified at the args flag:

redis = app.DB.use(RedisStorage, port=6719, host="localhost")
print(redis)

class express.helpers.RedisStorage(instance, **args)

Depending on availability of the redis module (from redis-py, will either create a new StrictRedis with conveinience wrappers or return nothing at all.

RedisStorage.IO(key, data)

Underlying method for setting the data into key, with locking so the writes and saves are atomic.

RedisStorage.get(key, default=None)

Gets the data found at key in the database, else it will return the default value, which defaults to None.

RedisStorage.gethash()

Get a unique hash based on the current time obtained from the time.asctime() method and hashed using SHA224. The unique hash is then checked for existence within the database.

If a key under the same hash exists (low probability), it will add a random character, either “y”, “e”, “o”, or “n” to the hash until it is no longer found within the DB.

RedisStorage.keys()

Return the keys within the database. Proxy method for StrictRedis.keys()

RedisStorage.rem(key)

Removes the key from the database and then performs a save.

RedisStorage.save()

Perform a StrictRedis.bgsave() since this is much better than the save() counterpart because we don’t have to wait for the database to save before we continue serving the request.

RedisStorage.set(key, data, proc=False)

Set the data into key. If proc is set to True, it will return a multiprocessing.Process object that has been started. Else, it will return True once the IO operation has completed.

class express.helpers.DBMBinding(instance)

Binds using the imported dbm implementation, in this order depending on availability:

  • gdbm (no maxsize-4KB restriction)
  • ndbm (faster than dbm.dumb implementation)
  • dumb (generally available on all platforms)
DBMBinding.open(filename, flag='r', mode=438)

Opens a dbm file which is a proxy for the open function of the loaded dbm module.

class express.helpers.SMTPBinding(instance, **kwargs)

SMTP binding for SMTP servers. You will need to configure it as you may not have a SMTP server running at localhost:0. Accepts an Express instance argument as it’s first function, but ignores it.

Other keyword arguments are passed directly to the smtplib.SMTP class/object which will serve as the client for you.
